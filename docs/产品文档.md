下面是一份把我们前面所有讨论**完整合并**后的 MVP 清单（不依赖硬件、以场景记录为核心、iOS 风格、Flutter 跨端），并包含**可直接做原型/开发的交互流程**。

---

## 0) MVP 定义

### MVP 目标（成功标准）

- **10 秒内**完成一次“物品位置记录”（选物品 → 拍照即保存）
- **30 秒内**在“找不到”时打开该物品**最后一次记录**并获得线索（照片/地点/标签）
- 轻提醒做到：**不强迫、不计时、可忽略**，每天 1–2 次即可带来“顺手推进一点点”的感觉

### MVP 边界（暂不做）

- 不做蓝牙/硬件/断连提醒/实时定位
- 不做番茄钟/计时/强任务管理
- 不做复杂 AI 自动识别（可留接口）

---

## 1) 信息架构（底部 4 Tab）

1. **物品**（Items）
2. **场景**（Scenes：回家/到公司/下车/旅行/临时收纳）
3. **顺手**（Gentle Nudges：轻提醒 + 意图池）
4. **我的**（Stats/Settings/Permissions）

> iOS 风格建议：大标题 + 分组卡片 + 明确主按钮（Primary CTA），尽量“一屏完成”。

---

## 2) MVP 功能模块清单（可直接拆任务）

### A. 物品库 Items

- [ ] 物品列表（搜索）
- [ ] 新建物品：名称 / 类别 / 重要程度 / 封面（默认图标即可）
- [ ] 物品详情页：
  - [ ] **最后一次记录**置顶（大图优先）
  - [ ] 记录时间线（按时间倒序）
  - [ ] 主按钮：**记录一次**
  - [ ] 次按钮：**我找不到了**

### B. 场景 Scenes（场景化记录入口）

- [ ] 内置场景模板：**回家 / 到公司 / 下车停车 / 旅行入住 / 临时收纳**
- [ ] 场景配置：
  - [ ] 默认要记录的物品集合（勾选）
  - [ ] （可选）该场景是否启用轻提醒联动

### C. 记录 Records（核心）

- [ ] **两步记录（强制）**：选物品 → 拍照 → **自动保存**
- [ ] 自动写入：时间戳
- [ ] 可选写入：定位（允许则存，不允许也可用）
- [ ] 保存后可补全（不打断流程）：
  - [ ] 备注（一句话）
  - [ ] 标签（玄关/抽屉/背包/车里…）
  - [ ] 场景（默认来自入口，可修改）

### D. 找回 Retrieve（核心）

- [ ] “我找不到了”模式：
  - [ ] 最后一次记录大图 + 时间 + 地点（有则显示）+ 标签/备注
  - [ ] 查看上一条/下一条（防止最后一次不准）
  - [ ] （有坐标时）打开地图/导航
  - [ ] **找到后更新**（拍照生成新记录）

### E. 出门检查 Check（减少忘带）

- [ ] “出门模式”一键入口
- [ ] 显示本次清单（来源：用户选择/场景默认）
- [ ] 快速勾选（可长按跳去该物品最后记录）
- [ ] 完成按钮：结束并记一次“出门事件”（用于统计）

### F. 轻提醒 + 意图池 “顺手”（替代传统防拖延、无计时）

**意图池（不是待办清单）**

- [ ] 新增意图：一句话“想推进的事”
- [ ] 可选：下一小步（10 秒能写）
- [ ] 可选：标签（在家/到公司/周末/通勤）用于更贴合的提醒

**轻提醒（Gentle Nudges）**

- [ ] 每天 0–3 次可配置（默认 1–2 次）
- [ ] 舒适时段可配置（例如中午/晚上）
- [ ] 触发来源：
  - [ ] 定时（MVP 建议先做这个，最稳）
  - [ ] 场景顺带（完成“回家记录包/出门检查”后在 App 内出现一条轻卡片）

**一屏完成卡片**

- [ ] 卡片展示：意图标题 + 下一小步（没有就引导“写一个最小下一步”）
- [ ] 三按钮（必须可忽略）：
  - [ ] **顺手做一点**（勾掉下一小步 / 或编辑更小）
  - [ ] **换一个**
  - [ ] **先不**（今天不再提醒/稍后）

**文案语气随机（温柔 + 俏皮）**

- [ ] 混合随机：默认 温柔 70% / 俏皮 30%
- [ ] 夜间更温柔（例如 21:00–08:00 温柔 85%）
- [ ] 去重：最近 10 条不重复（同一意图 24h 内不重复同一句）

### G. 我的 Me（统计 + 设置）

- [ ] 统计：本周记录次数 / 找回次数 / 出门检查次数 / 意图推进次数
- [ ] 权限入口：相机/定位/通知
- [ ] 提醒设置：频率、时段、语气（温柔/俏皮/混合）

## 4) 核心交互流程（可直接做流程图）

### 流程 A：首次使用 Onboarding

1. 欢迎页（价值一句话）
2. 勾选常用物品（钥匙/门卡/耳机/证件…）
3. 勾选常用场景（回家/到公司/下车/旅行…）
4. 权限分步请求：先相机；通知/定位在需要时再要（更像 iOS 好应用）

---

### 流程 B：场景化记录（回家/到公司/下车）

入口：场景 Tab → 选择“回家”

1. 展示“本场景推荐记录的物品”卡片列表
2. 点某物品 → 相机
3. 拍照后**自动保存**（toast：已保存）4)（可选）补全：标签/备注/定位展示
4. 返回场景页 → 继续下一个物品（支持连续拍）

> 关键：**拍完即保存**，补全不强制。

---

### 流程 C：快速记录（不走场景）

入口：物品 Tab → 物品详情 → “记录一次”

1. 相机 → 拍照自动保存
2. 可选补全：标签/备注/场景

---

### 流程 D：找不到（找回模式）

入口：物品详情 → “我找不到了”

1. 大图展示最后一次记录
2. 显示时间/地点（若有）/标签/备注
3. 操作：查看上一条 / 打开地图 / 找到后更新（拍照写新记录）

---

### 流程 E：出门检查（减少忘带）

入口：场景 Tab → “出门模式”

1. 展示本次清单（可编辑本次）
2. 逐个勾选（可长按查看该物品最后记录）
3. 点“完成出门” → 记录一次事件（用于统计）

---

### 流程 F：“顺手”轻提醒（无计时、可忽略）

**定时通知**：每天 1–2 次

1. 通知文案（温柔/俏皮随机，且去重）
2. 点开进入“今日一张卡片”：
   - 顺手做一点 / 换一个 / 先不（今天不再提醒）

3. 顺手做一点：
   - 勾掉下一小步 ✅ 或改成更小一步（不计时、不评价）

**场景顺带**（更自然）：

- 完成“回家记录包/出门检查”后，App 内底部出现一条轻卡片（可直接忽略）

## 5) 最小数据结构（开发会很顺）

- **Item**：id, name, category, importance, coverImage
- **Scene**：id, name, icon, defaultItemIds[]
- **Record**：id, itemId, sceneId?, photoUri, timestamp, location?, placeName?, note?, tags[]
- **Intent**：id, title, nextStep?, tags[]（home/work/weekend/commute）, updatedAt
- **NudgeHistory**（本地）：recentTemplateIds[10], lastUsedByIntent（可选）

---

## 6) Flutter MVP 技术要点（简版）

- UI：`CupertinoApp + CupertinoPageScaffold` 做 iOS 质感
- 存储：
  - MVP：`shared_preferences`（recent 文案、用户设置、轻量列表）
  - 记录照片/数据量起来后：再换 `sqflite/isar/hive`（第二阶段）

- 通知：`flutter_local_notifications + timezone`
- 文案库：`assets/nudges.json`（温柔/俏皮 + 场景标签），随机选择时做**去重冷却**

- [ ] 出门模式：能勾选清单并完成一次出门事件
